# 競合ベンチマーク

| Competitor / Standard | 強み | 採用する基準 | 差別化 |
| --- | --- | --- | --- |
| Chrome DevTools | 既存ユーザーが多い | 基本操作の短さ | 拡張開発で必要な content script、selector、message flow の確認を小さく束ねる。 |
| GitHub Release | 配布と証跡が残る | release notes と assets | docs ZIP と manual-test を必須化 |
| Local CLI validation | 再現性が高い | representative suite | ホストUIより先に自動検証を安定化 |

QCDS 評価では、競合の機能数ではなく、closed alpha として再現可能な証跡、手順、手動確認の明示を重視します。
